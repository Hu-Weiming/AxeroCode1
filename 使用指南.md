# AxerCode 使用指南

## 先决条件

- 本机已安装并运行 `Ollama`
- 本机模型里有 `qwen2.5:7b`
- 推荐先确认：

```powershell
ollama list
```

如果没启动 Ollama，可以先执行：

```powershell
ollama serve
```

- [axercode-cli.exe](/D:/AeroCode1/axercode-cli/target/axercode-cli.exe)

D:\AeroCode1\axercode-server\target\axercode-server.exe

- [http://127.0.0.1:8080/](http://127.0.0.1:8080/)

## 这个 Agent 怎么用

AxerCode 的核心使用方式分两类：

1. 普通输入  
直接说需求，例如：
`请帮我分析这个目录`
`请帮我写一个 Spring Boot 接口`
`请帮我比较这两个实现方案`

2. 斜杠命令  
适合管理上下文、分支、计划和检查点。

常用命令：

- `/help`：查看命令帮助
- `/history`：查看当前会话历史
- `/new`：开始一个新会话
- `/status`：查看当前状态
- `/plan on`：开启计划模式
- `/plan off`：关闭计划模式
- `/focus <路径>`：把注意力聚焦到某个目录
- `/focus clear`：清除 focus
- `/checkpoint <名字>`：保存当前检查点
- `/checkpoints`：查看检查点列表
- `/restore <名字>`：恢复检查点
- `/diff`：比较当前会话和检查点/分支差异
- `/branch <名字>`：创建或切换会话分支
- `/branch list`：查看分支列表
- `/exit`：退出

## 推荐的基本用法

### 场景 1：先分析再动手

```text
/plan on
/focus D:\AeroCode1
请先分析当前模块结构，再给我一个最小改动方案
```

### 场景 2：做改动前先留快照

```text
/checkpoint before-refactor
请重构当前 CLI 启动流程
```

### 场景 3：做实验性尝试

```text
/branch experiment-native
请尝试优化 native 启动时间
```

## 使用技巧

- 任务尽量具体  
比起“帮我改一下”，更推荐：
`请把 axercode-cli 的启动参数整理成更清晰的结构`

- 路径尽量明确  
如果任务和某个目录强相关，先用 `/focus`。

- 大任务先开计划模式  
如果是重构、迁移、构建新模块，先 `/plan on`。

- 重要改动前先做 checkpoint  
这样回滚和对比都更轻松。

- 实验性思路尽量走 branch  
不要把所有尝试都混在同一个会话里。

- 一次只给一个核心目标  
模型在“一个任务一个目标”时最稳定。
